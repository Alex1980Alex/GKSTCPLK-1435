Группа общие

Команда Серверы очередей RMQ

Группа Исходящие

Команда Исходящие сообщения  - Предназначена для открытия формы списка справочника "Исходящие сообщения". Сообщения внутри транзакций не отправляются немедленно, а записываются в специальный справочник «ИсходящиеСообщения». Затем, эта очередь исходящих сообщений рассылается регламентными заданиями. В результате, запись сообщения в справочник будет происходить в рамках вышестоящей транзакции и в случае сбоев будет откатываться вместе с вышестоящей транзакцией.

Команда Задания на отправку - Предназначен для настройки заданий на отправку исходящих сообщений. В справочнике указывается количество потоков, которые будут заниматься рассылкой накопленных сообщений и расписание их работы. Кроме того, имеется возможность разделить обработчики рассылки по конкретным серверам очередей или точкам публикации. Для этого на закладке «Отбор» можно задать фильтр на соответствующие поля справочника исходящих сообщений». Обработчик будет выбирать только сообщения, соответствующие отбору. Технически, каждый поток-обработчик представляет собой регламентное задание, работающее по заданному расписанию. Состояние выполнения всех заданий можно увидеть на закладке «Состояние обработчиков».

Группа Входящие

Команда Подписки на события - Предназначен для создания «прослушивающих» конкретную очередь регламентных заданий. Каждый элемент справочника создает регламентное задание, работающее по алгоритму прослушивания очереди (см. выше). Для каждой подписки задается: 
- Сервер, на котором находится очередь 

- Имя очереди Интервал ожидания событий в очереди 

- Расписание старта задания

По истечению интервала прослушки, если за это время не поступило никаких сообщений, задание завершает свою работу и в следующий раз снова стартует по расписанию. Данный справочник удобно заполнять с помощью механизма «Создания на основании». На основании элементов справочника «ОбработчикиСобытийRMQ» могут быть созданы подписки на очереди.

Команда Обработчики событий - После регистрации обработчиков в коде приложения, в режиме предприятия станет возможно привязывать тот или иной обработчик с физической очередью сервера RabbitMQ. Для олицетворения процедур-обработчиков в режиме 1С:Предприятия используется справочник «Обработчики событий». Этот справочник не предполагает ручного заполнения, его элементы создаются на основании зарегистрированных процедур-обработчиков.
